A — Project × week grid (custom LWC)RECOMMENDED
Pros✓ The Excel grid, inside Salesforce — all 6 weeks on one screen
✓ Self-heals missing records at 0 hrs, span-bounded
✓ Live capacity row + non-blocking warning
✓ Delegate mode covers contractors
Cons✗ Largest single build
✗ New component to own
Blue cells are editable. Rows appear automatically for every active assignment — nothing to add manually.
B — Enhance the existing Forecast Entry screen
Pros✓ Smallest change, fastest to ship
✓ Fixes phantom-hours bug + inline editing
✓ Missing rows auto-created at 0
Cons✗ Still one week at a time
✗ Doesn't hit "6 weeks in under 5 minutes"
Pill switcher replaces the date picker; each edit saves the row instantly.
C — Capacity Canvas: allocate hours into weekly budgets
Pros✓ Capacity-first — overbooking visible the instant it happens
✓ Soft color feedback (green → amber → red)
✓ Tap-friendly, no typing needed
Cons✗ New mental model vs the familiar spreadsheet
✗ Slower than the grid for bulk entry
Each card is one week. The bar is capacity; every + fills it. Amber at 90%, red when over — saving still allowed.
D — Team Heatmap: the manager's cockpit
Pros✓ Whole team × 6 weeks on one screen — spot gaps & overloads instantly
✓ Click any cell to edit that person's week without leaving the page
✓ Doubles as the Objective 4 heatmap later — one component, two uses
Cons✗ Manager-first; individuals still need A/B/C-style entry for themselves
✗ Densest screen — needs training for occasional users
Cell = person's forecast total / capacity for the week. Green = healthy, yellow = light (under-forecast), red = overbooked, grey = no records yet. Click a cell to edit its project breakdown.
E — Smart Confirm: propose, review, done
Pros✓ Fastest possible entry — system proposes the 6 weeks from your recent pattern, you accept or tweak
✓ Perfect for steady managed-services work (most of Blue Ocean's rows never change)
✓ One click for the common case; typing only for exceptions
Cons✗ Proposals are only as good as history — new projects still need manual entry
✗ Risk of rubber-stamping without thinking
✗ Needs A/B-style fallback editing for corrections
Proposed hours = copy of your last populated week (later: 4-week average). Accept to confirm, or Adjust to fine-tune just that project.
